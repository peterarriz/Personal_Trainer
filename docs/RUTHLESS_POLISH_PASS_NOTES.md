## Ruthless Polish Pass

This pass tightened the final layer of trust and presentation after the structural work was already in place.

### Core rules

- Prefer product language over system language.
- Explain capability once, then get out of the way.
- Keep choices explicit without sounding defensive or operational.
- Use the same words everywhere for the same concept.

### Language shifts

- `custom fallback` -> `custom goal`
- `Still fuzzy` -> `Still open`
- `No hard end date` -> `No fixed deadline`
- `Local-only fallback` / `Device-only mode` -> calmer device-first labels
- `Reset plan only` -> `Reset plan`
- `Validate restore code` -> `Review restore`

### UI cleanup targets

- Auth now frames the local/cloud choice in product terms instead of failure-mode terms.
- Intake now treats custom entry as a real path, not a degraded fallback.
- Settings Goals no longer leads with migration or admin-panel wording.
- Goal cards and review cards no longer expose internal-sounding `fuzzy` language.
- Loading state now feels like a product surface instead of a placeholder.

### What stayed intentionally visible

- Delete-account diagnostics still expose required deployment information when delete is unavailable.
- Goal previews and audit details still exist, but their surrounding copy is calmer and less bureaucratic.
- Open-ended goals remain explicit about the visible horizon without implying a fake deadline.
