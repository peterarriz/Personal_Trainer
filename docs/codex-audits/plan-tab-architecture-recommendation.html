IA decision

Plan should become the home for why the block looks the way it does.

The current ownership is split between `Program`, `Settings > Plan style`, `Settings > Goals`, and baseline repair seams. The question is whether a new `Plan` tab should sit beside `Program`, or whether it should replace it.

Selected direction

Plan absorbs Program.

Adding `Plan` while keeping `Program` creates two adjacent tabs that both mean "the plan." The cleaner product is `Today / Plan / Log / Nutrition / Coach`, where Plan owns both the block narrative and the controls that shape it.

Two options

One keeps the split. One resolves it.

Option A preserves the current Program reading surface and adds a new Plan tab for ownership controls. Option B lets Plan absorb Program so the user has one place to both understand and shape the block.

Option A

Clean on paper, messy in the shell.

  • `Program` still means "the plan" to most users, even if the IA says it only reads the block.
  • Adding `Plan` beside it creates two adjacent plan tabs.
  • The split is safer architecturally, but weaker product-wise.

Option B

The better product model.

  • `Today` tells you what to do. `Plan` tells you why the plan looks like this.
  • The user no longer has to remember whether plan controls live in Program or Settings.
  • The shell stays at five strong tabs instead of growing to six overlapping ones.
  • This is the better migration target even though it is the bigger refactor.
