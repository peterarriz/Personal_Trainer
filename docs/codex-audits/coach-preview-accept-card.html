Hierarchy

Impact should do the persuasive work.

  • Headline: names the move.
  • What changes: leads with delta plus preserved anchor.
  • Why: supports the call in a quieter voice.
  • Commit: makes scope and consequence explicit before the action.
Optionality

Consequence should disappear rarely.

Only omit the consequence block when there is no honest numeric delta, no preserved anchor worth surfacing, and the headline already contains the full outcome.

If inventing precision would be faker than staying quiet, omit it. Otherwise consequence stays mandatory.

Commit language

The final action should feel binding.

  • Use Commit, not Accept.
  • Name the scope when possible: today, weekly, suggested change.
  • Let the secondary action defend the status quo: Keep current plan.
