# Programs Vs Styles Contract

## Purpose

Programs and Styles both influence planning, but they are not the same thing.

## Program

A Program is a concrete multi-week backbone.

A Program can change:

- weekly structure
- session family allocation
- progression expectations
- fidelity rules
- compatibility status

A Program must not:

- replace the goal stack
- bypass hard safety rules
- silently claim perfect literal fidelity when compatibility is weak

## Style

A Style is a soft overlay.

A Style can change:

- conditioning slant
- exercise flavor
- emphasis and feel
- optional secondary work

A Style must not:

- become the whole weekly backbone
- override hard Program rules
- replace goal ownership

## Fidelity Modes

- `adapt_to_me`: Program is the backbone, but the live planner can adapt structure to the athlete
- `strict`: Program backbone is preserved more literally until compatibility or adherence forces a downgrade
- `use_as_style`: Program definition is treated as a style-like overlay, not the full backbone

## Compatibility Rules

- compatible Program: materially reshapes the real week
- incompatible Program: blocked or downgraded with a plain-English reason
- compatible Style: changes emphasis without pretending to be the backbone
- incompatible Style: rejected without partial silent bleed-through

## User-Facing Expectations

- If a named Program is active and compatible, the week should visibly look different.
- If a Style is active, the week should feel different but still follow the same backbone.
- If the Program is downgraded, the user should see that in the basis explanation.
- If the Program is only being used as a style, the app must say so.

## Correct Behavior Examples

- `Half Marathon Base` as Program:
  - running sessions reshape the week skeleton
  - the Program becomes the visible backbone

- `Fight Camp Lean` as Style:
  - conditioning flavor changes
  - the primary weekly backbone still comes from the goals or Program

- `Powerbuilding Builder` in `use_as_style` mode:
  - strength flavor changes
  - the planner does not pretend the full powerbuilding template is running as written
