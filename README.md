# FORMA

FORMA is a personal training app that combines planning, logging, nutrition, check-ins, goals, and coach support in one product.

## Repo Shape

This is not a single-file app anymore.

- `src/trainer-dashboard.jsx` is the client entry point and composition root.
- `src/services/` holds most domain logic, product view models, and state adapters.
- `src/modules-*.js` contains older shared modules that are still part of the runtime.
- `api/` contains serverless endpoints for auth, AI, and device integrations.
- `tests/` contains the Node-based service and contract test suite.
- `e2e/` contains Playwright product flows.
- `docs/` contains product specs, audits, and design notes.

The deployed app is still generated into the root `index.html`, but that file is build output, not the source of truth.

## Stack

- React 18
- Supabase auth and persistence
- Vercel serverless functions
- Node test runner plus Playwright

## Common Commands

```bash
npm install
npm run build
npm test
npm run e2e
npm run dev
```

## Workflow

1. Edit source in `src/`, `api/`, or `scripts/`.
2. Run the smallest relevant test pass first.
3. Run `npm run build` to regenerate the deployable `index.html`.
4. Verify that any `index.html` diff is build output only.

## Generated Output

- Root `index.html` is generated by `scripts/build.js`.
- Local QA and theme-capture output belongs in `artifacts/`.
- `dist/`, `tmp/`, `playwright-report/`, and `test-results/` are disposable output.

## Deployment

Vercel is configured to serve the repo root, so the build currently emits to `./index.html`.
